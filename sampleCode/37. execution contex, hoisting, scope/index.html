Phase execution contex
  1. Creation
  2. Execution
Pada saat program dijalankan ada sesuatu yang terjadi yang dinamakan Creation Phase yang terjadi pada context global jika dibuat langsung didalam file javascriptnya/window (bukan didalam function)
    Proses Creation Phase (Phase pembentukan) :
  1. Cek apakah ada function atau variable didalam windownya
  2. Jika ada var akan diisi dengan undefined secara default ex : var nama = undefined
  3. jika ada function akan diisi dengan function itu sendiri ex : function nama = fn()
    4. konsep diatas dinamakna Hoisting(menggeret)

    ex :

    console.log(nama)

    var nama = 'Mimin'

    Pada dua baris code diatas, var akan digeret keatas dengan nilai undefined dan console log tidak dijalankan, karena jika ada kode console.log(nama) saja tanpa ada var nama = 'Mimin' akan terjadi eror dengan keterangan nama tidak dikenali

selain mendefinisikan dua hal tadi (var atau function) javascript juga mendefinisikan object window sebagai global object dan this sebagai window, untuk ngeceknya ketikan window atau this pada halaman console maka akan tampil beberapa object meskipun didalam file javascript tidak ada baris code apapun
    Proses execution phase:
  1. Mengeksekusi program dari atas kebawah

    ex :

    var nama = 'Yana'

    umur = 34

    console.log(`Halo nama saya ${nama}, umur saya ${umur}`)

    }
